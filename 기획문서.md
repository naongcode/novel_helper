# Novel Helper — AI 소설 창작 도우미 기획문서

## 1. 프로젝트 개요

소설을 쓸 때 가장 어려운 세계관 구성, 캐릭터 설계, 줄거리 계획을 체계적으로 관리하고, AI가 자동으로 생성·보조해주는 웹 애플리케이션.

---

## 2. 핵심 기능 요약

| 기능 | 설명 |
|------|------|
| 다중 프로젝트 | 여러 소설 프로젝트를 독립적으로 관리 |
| 작품 설정 | 가치관·엔딩·분위기·문체 설정 + AI 자동생성 |
| 세계관 관리 | 종족·나라·화폐·기타 섹션별 자유 편집 + AI 자동생성 |
| 캐릭터 관리 | 캐릭터 프로필·관계도 + AI 자동생성 |
| 줄거리 계획 | 챕터별 요약·핵심사건 + AI 전체 줄거리 생성 |
| 본문 작성 | 장면 단위 편집 + AI 장면 생성 |
| 버전 관리 | 장면 생성 시 git 자동 커밋으로 이전 버전 보존 |
| JSON 파일 저장 | 프로젝트 도메인별 JSON 파일로 로컬 저장 |

---

## 3. 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js 15 (App Router) | API Routes로 OpenAI 키 서버사이드 보호 |
| 언어 | TypeScript | 타입 안전성 |
| 스타일 | Tailwind CSS v4 | 빠른 UI 구성 |
| AI | OpenAI GPT-4o | 한국어 창작 품질 |
| 저장소 | JSON 파일 (data/ 폴더) | 도메인별 파일 분리, AI 프롬프트 조립 용이 |
| 버전 관리 | Git 자동 커밋 | 장면 생성 이력 보존 |
| 상태관리 | React useState/useCallback | 별도 라이브러리 불필요 |

## 5. 데이터 구조 (JSON 파일)

### 파일 저장 구조

```
novel_helper/
└── data/
    └── {id}/
        ├── meta.json          # 제목, 장르, 작품 설정, 날짜
        ├── world.json         # 세계관 4섹션
        ├── characters.json    # Character[] 배열
        └── chapters/
            └── {chid}.json    # 챕터 정보 + Scene[] 포함
```

### 파일 API 라우트 (CRUD)

| 엔드포인트 | 동작 |
|-----------|------|
| `GET /api/projects` | `data/` 하위 폴더 목록 + 각 `meta.json` 반환 |
| `POST /api/projects` | `{id}/` 폴더 생성, 파일 초기화 |
| `DELETE /api/projects/[id]` | `{id}/` 폴더 전체 삭제 |
| `GET /api/projects/[id]/world` | `world.json` 반환 |
| `PUT /api/projects/[id]/world` | `world.json` 저장 |
| `GET /api/projects/[id]/characters` | `characters.json` 반환 |
| `PUT /api/projects/[id]/characters` | `characters.json` 저장 |
| `GET /api/projects/[id]/chapters` | `chapters/` 전체 반환 |
| `PUT /api/projects/[id]/chapters/[chid]` | `chapters/{chid}.json` 저장 |
| `DELETE /api/projects/[id]/chapters/[chid]` | `chapters/{chid}.json` 삭제 |
| `PUT /api/projects/[id]/meta` | `meta.json` 저장 (작품 설정 포함) |

### 타입 정의

```ts
interface ProjectConcept {
  theme: string    // 가치관/주제
  ending: string   // 엔딩 방향
  tone: string     // 분위기/톤
  style: string    // 문체
}

interface ProjectMeta {
  id: string
  title: string
  genres: string[]    // 복수 선택, 빈 배열 허용 (예: ["무협", "로맨스"])
  concept: ProjectConcept
  createdAt: string   // ISO 8601
  updatedAt: string
}

// 선택 가능한 장르 목록 (상수)
const GENRE_OPTIONS = [
  "판타지", "무협", "로맨스", "SF", "추리",
  "공포", "역사", "현대", "성장", "기타"
]

interface WorldSetting {
  races: string
  countries: string
  currencies: string
  misc: string
}

interface Character {
  id: string
  name: string
  role: string
  personality: string
  background: string
  appearance: string
  appearanceChapter: number | null
  relationships: {
    targetId: string
    targetName: string
    relation: string
    description: string
  }[]
}

interface Scene {
  id: string
  order: number
  description: string  // 장면 설명 (AI 생성 지침)
  content: string      // AI가 생성한 본문
}

interface Chapter {
  id: string
  number: number
  title: string
  summary: string
  keyEvents: string[]
  scenes: Scene[]
}
```

## 6. AI API 설계

### 공통: SSE 스트리밍

```
data: {"type": "chunk", "content": "..."}\n\n
data: {"type": "done"}\n\n
data: {"type": "error", "error": "..."}\n\n
```

### AI 컨텍스트 규칙

| 데이터 | 전달 방식 |
|--------|----------|
| 작품 설정 (concept) | 항상 전문 |
| 세계관 (4섹션) | 항상 전문 |
| 캐릭터 전체 | 항상 전문 |
| 해당 챕터 줄거리 | 전문 (제목·요약·핵심사건·장면 설명 목록) |
| 다른 챕터 줄거리 | 요약 1줄씩만 |
| 완성된 본문 (content) | 전달하지 않음 |

### 엔드포인트

| 엔드포인트 | 입력 | 출력 |
|-----------|------|------|
| `POST /api/generate/concept` | 제목, 장르 | concept JSON (스트리밍→파싱) |
| `POST /api/generate/world` | 섹션명, 시드, concept, 나머지 3섹션 | 텍스트 (스트리밍) |
| `POST /api/generate/character` | 이름, 역할, concept, 세계관, 기존 캐릭터 | 캐릭터 JSON (스트리밍→파싱) |
| `POST /api/generate/plot` | 챕터수, concept, 세계관, 캐릭터 | 챕터 JSON 배열 (스트리밍→파싱) |
| `POST /api/generate/scene` | 장면설명, concept, 세계관, 캐릭터, 해당챕터, 다른챕터요약 | 텍스트 (스트리밍) |

## 7. Git 기반 버전 관리

```
장면 생성 → chapters/{chid}.json 저장 → git commit 자동 실행
```

- 커밋 메시지 예: `[검은탑의마법사] 1챕터 2장면 생성`
- 이전 버전 복원: git log → git checkout

```
챕터 1
  장면 1 [완료] ── commit ──┐
  장면 2 [완료] ── commit ──┤ → 언제든 되돌리기 가능
  장면 3 [생성중]           │
```
